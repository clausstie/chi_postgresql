Windows Instructions

    Instructions

    1. After downloading, double-click on marvinbeans-4.0.5-win_novm.exe or marvinbeans-4.0.5-win_vm.exe.
    2. After the installation, you can add the bin folder of Marvin Beans to the PATH to be able to run Marvin applications from any directory in the command line. For more details about the editing the PATH, please consult the Windows Help.

    Notes

    • If you do not have a Java installed, be sure to download the package above which includes it.

Mac OS X Instructions

    Instructions

    • After downloading, double-click marvinbeans-4.0.5-osx.zip.
    • After the installation process, add the bin folder of the Marvin Beans folder to the PATH to be able to run Marvin applications from any directory in command line.

    Notes

    • Requires Mac OS X 10.0 or later
    • The compressed installer should be recognized by Stuffit Expander and should automatically be expanded after downloading. If it is not expanded, you can expand it manually using StuffIt Expander 6.0 or later.
    • If you have any problems launching the installer once it has been expanded, make sure that the compressed installer was expanded using Stuffit Expander. If you continue to have problems, please contact technical support.

Linux Instructions

    Instructions

    1. After downloading, open a shell and cd to the directory where you downloaded the installer.
    2. At the prompt type: ./marvinbeans-4.0.5-linux.bin
    3. After the installation, add the bin subdirectory of the Marvin Beans directory to the PATH to be able to run Marvin applications from any directory.

    Notes

    • You need to install a Sun's Java Runtime Environment 1.3.1 (or later). You can download it from Sun's Java web site or contact your OS manufacturer.

    Troubleshooting

    • The installer stops with the following message: No Java virtual machine could be found from your PATH environment variable.
      Cause: There is no (suitable) Java on your machine or the java command is missing from the PATH environment variable.
      Solution: Install a Java Runtime Environment from Sun if it is not installed. Then put <JAVAHOME>/bin to the PATH where <JAVAHOME> is the home directory of Java. For more details about setting environment variable, please consult the manual of the current shell. Then restart the installer.

    • The installer is failed with any other error message.
      Cause: Installation may fail if other distributor's Java is (also) installed (e.g. kaffe-1.0.6-6 or jdkgcj-0.2.3-5).
      Solution: Please check if non-Sun distributed Java Runtime Environment is installed: 

      rpm -qa |grep kaffe
rpm -qa |grep jdkgcj
      If one of them is installed, remove that package (administrator permissions are required). E.g.: 
      rpm -e jdkgcj
      After this, install a Sun Java Runtime Environment 1.3.1 (or later) if it is not installed yet. Then try to relaunch the installer.

    • If you have further problems with the installer use marvinbeans-4.0.5.zip.

Instructions for All Other Platforms

    Instructions

    • After downloading go to the directory where you downloaded marvinbeans-4.0.5.zip then uncompress the zip file.
    • You can start applications by script or batch files, which you can find in the marvinbeans/bin directory.

    Notes

    • Requires Java 1.3.1 (or later). You can download it from Sun's Java web site or contact your OS manufacturer.
    • You need an expander which can handle zip extension.
    • Batch files (bin/*.bat) have to be initialized before first use. Edit files to set the MARVINBEANSHOME variable to the full path of the directory where Marvin Beans is located.

Howto uninstall

    Use the uninstaller to remove Marvin Beans from your machine.

    Windows and Mac OS X

    Double click on the mb_uninst or mb_uninst.exe icon in the Uninstall subfolder of Marvin Beans' home folder.

    Linux

    Launch the mb_uninst script in the Uninstall subdirectory of Marvin Beans' home directory.
